## Technical Concepts to Explain (Paul's Way)

### Postage Stamp Technique
From Moondream team - create a grid of frames over time so single-frame VLM can interpret temporal changes.

### Cloud vs Local Trade-offs
- Cloud: Easy testing, low compute
- Local: HIPAA compliant, maximum privacy, free for deployment

### The Harness
Industry-specific guidelines that make AI coding tools work better for ProAV/Broadcast.

### Scalability Difference
- CV: Build unique model for each application (horses, basketballs, etc.)
- VLM: Track anything with the same model

## Writing Guidelines

### Do

```
- Start with a story or "wow" moment
- Name specific people and give credit
- Include exact numbers (5,000 API calls, 20 years)
- Explain WHY something matters (scalability, cost, privacy)
- Address the reader directly
- Mention trade shows where you meet readers (IBC, NAB, InfoComm)
- Talk about the "movement" not just the product
- Include both business and personal use cases
```

### Don't

```
- Use passive voice when active is clearer
- Start with abstract definitions
- Forget cost/practical considerations
- Claim solo credit (always "we" for team efforts)
- Use corporate buzzwords
- Skip the human story
- Forget to mention free resources (YouTube courses, free API tier)
```
